Tell us the use cases of null and undefined

NULL:Null is often used to explicitly represent the absence of any object value or as a placeholder for an object that doesn't exist or is not applicable. Example :If the user is not logged in, you could assign the value null.

UNDEFINED:Undefined indicates that a variable is not assigned a value or not declared. If a function doesn't have an explicit than the return statement returns undefine by default. When a function has optional parameters, and if a parameter is not provided, it is often left undefined. This allows the function to check whether a specific parameter was passed or not.

What do you mean by REST API?

RESTful APIs, short for Representational State Transfer Application Programming Interfaces, are a widely used architectural style for designing web services. They leverage standard HTTP methods like GET, POST, PUT, and DELETE to facilitate communication and data exchange between diverse software applications. Guided by principles such as statelessness, a uniform interface, and the use of resources identified by URIs, RESTful APIs have become a cornerstone of modern web development. Their standardized approach enables seamless integration and data sharing across various applications, fostering a more interconnected and dynamic digital ecosystem.
